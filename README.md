# 🧪 Toxic Ingredients Indicator

An AI powered Streamlit app that turns a cosmetic product label into a plain English safety report. Users can type or paste an ingredient list, or upload a photo of a label, and the app returns hazard tiered results for every flagged ingredient, along with a product level summary and per ingredient explanations generated by Gemini 2.5 Flash.

Built as the Applied Research Project for the MS in Business Analytics at California State University, East Bay.

## What It Does

Cosmetic ingredient lists are long, dense, and written in INCI (International Nomenclature of Cosmetic Ingredients) Latin terminology. Regulators hold curated risk data, manufacturers hold formulation knowledge, and consumers hold neither. This app closes that gap.

Users can interact with the system in three ways:

* **Search** a single ingredient or CAS number
* **Paste** a full ingredient list copied from a product page
* **Upload** a photo or PDF of a physical label, with an in browser crop and rotate tool

The app then parses the input, matches ingredients against a curated hazard database, sorts them by severity (High, Medium, Low), and generates a product level summary plus an on demand plain English explanation for each flagged ingredient.

---

## Key Features

* Curated database of **736 flagged ingredients** across **30 chemical families** and **5 severity tiers**
* Regulatory sources: **EU Cosmetics Regulation (EC No 1223/2009)**, **California AB 2762**, **IARC monographs**, **CIR Expert Panel reports**, **FDA cosmetic guidance**
* **Dual path OCR pipeline**: Tesseract for clean labels, Gemini Vision for dark, curved, or compressed photographs. The system automatically selects the higher quality output
* **15 preprocessing variants** (CLAHE, gamma correction, unsharp masking, deskew, contrast stretching, denoise) across 3 Tesseract PSM modes for the heavy OCR path
* **Two stage search**: SQLite FTS5 prefix matching with a LIKE based fallback, plus domain expansion for umbrella terms like "fragrance" and "paraben" that surface hidden allergens
* **Four Gemini 2.5 Flash features**: Vision OCR, OCR error correction, product summary, per ingredient explanation. All with constrained prompting, retry logic, and session state caching
* **Traceable citations**: every ingredient card links back to its original regulatory source
* Graceful degradation when the Gemini API key is absent. Core database features keep working

## Architecture

The system is a single page Streamlit app backed by a local SQLite database and external generative AI APIs, organized into four layers:

```
┌─────────────────────────────────────────────────────────┐
│  Input Layer                                            │
│  Text search  │  Paste from label  │  Image/PDF upload  │
└────────────────────────┬────────────────────────────────┘
                         │
┌────────────────────────▼────────────────────────────────┐
│  Text Extraction Layer (images only)                    │
│  Dual path OCR: Tesseract + Gemini Vision               │
└────────────────────────┬────────────────────────────────┘
                         │
┌────────────────────────▼────────────────────────────────┐
│  Search and Retrieval Layer                             │
│  FTS5 prefix match → LIKE fallback → domain expansion   │
└────────────────────────┬────────────────────────────────┘
                         │
┌────────────────────────▼────────────────────────────────┐
│  Result Enrichment Layer                                │
│  Severity sort → risk rollup → Gemini summary & explain │
└─────────────────────────────────────────────────────────┘
```

### Database Schema

A normalized five table SQLite design separates ingredient identity from hazard classification and source attribution, allowing many to many relationships between ingredients, hazards, and regulatory sources.

| Table | Purpose |
|---|---|
| `ingredients` | One row per unique INCI name with CAS registry number |
| `hazards` | Hazard records with severity tier (0 to 4), code, category, description |
| `ingredient_hazards` | Junction table linking ingredients to hazards and sources |
| `ingredient_aliases` | Trade names, misspellings, abbreviations, alternate INCI forms |
| `sources` | Regulatory and scientific reference sources with hyperlinks |
| `ingredients_fts` | FTS5 virtual table, rebuilt at startup, `unicode61` tokenizer, prefix index 2 to 10 chars |

### OCR Pipeline

Image quality is evaluated on three metrics (pixel height, mean intensity, standard deviation) to route between a fast path (minimal preprocessing, Tesseract PSM 6/4/3 sweep) and a heavy path (15 preprocessing × PSM combinations, scored by a weighted sum of Tesseract confidence and an ingredient specific token count heuristic). When the Gemini Vision button is active, both engines run in parallel and the higher count output wins.

Postprocessing applies OCR character substitution fixes (0→O, 1→I/L, |→L), normalizes punctuation between all caps tokens to commas, drops noise heavy lines, and runs a 25 item regex dictionary of specific INCI corrections observed during testing.

### Severity Tiering

| Tier | Label | UI | Criteria |
|---|---|---|---|
| 4 to 5 | High Risk | Red badge | IARC Group 1/2A carcinogens, endocrine disruptors, EU/CA banned substances |
| 2 to 3 | Medium Risk | Yellow badge | Confirmed sensitizers, IARC Group 2B, persistent bioaccumulators |
| 0 to 1 | Low Risk | Green badge | Possible sensitizers, limited evidence of harm |
| None | Unknown | Gray badge | Present in database but insufficient data to assign a tier |

For ingredients with multiple hazard records, the displayed tier reflects the maximum severity across all linked hazards.

## Tech Stack

| Layer | Technology | Role |

| Frontend / UI | Streamlit 1.x | Web app framework, session state, card rendering |

| HTTP client | Python `requests` | Gemini API calls with API key header authentication |
| Config | Streamlit `secrets.toml` | Secure API key storage with env var fallback |

## Installation

### Prerequisites

* Python 3.10+
* Tesseract 4 installed at the system level
* Poppler (for PDF support)
* A Gemini API key (optional. The app runs without it, AI features disabled)

### Setup

```bash
git clone https://github.com/SamadritaR/Toxic-Ingredients-Indicator-for-Beauty-Products.git
cd Toxic-Ingredients-Indicator-for-Beauty-Products

python -m venv .venv
source .venv/bin/activate          # Windows: .venv\Scripts\activate

pip install -r requirements.txt
```

### Configure the Gemini API key

Create `.streamlit/secrets.toml`:

```toml
GEMINI_API_KEY = "your key here"
```

Or set it as an environment variable:

```bash
export GEMINI_API_KEY="your key here"
```

### Run

```bash
streamlit run app.py
```

The app binds to `0.0.0.0:8501` by default, so you can access it from any device on the same WiFi network. Useful for photographing a label on your phone and viewing results on your laptop, or vice versa.

## How It Works

### Search and Retrieval

The search model tokenizes input on commas, semicolons, forward slashes, and newlines, then runs each candidate term through FTS5 prefix matching ranked by BM25. Terms that return no FTS hits fall back to a case insensitive LIKE query across both the ingredient names and the alias column.

Two domain expansion rules fire before deduplication: inputs containing `fragrance` or `parfum` trigger automatic lookup of six high prevalence fragrance allergens (linalool, limonene, citral, eugenol, benzyl salicylate, benzyl benzoate), and inputs containing `paraben` trigger lookup of all six major paraben variants. These rules address the industry practice of listing generic umbrella terms on labels to conceal specific regulated chemicals.

### Constrained Prompting for Gemini

All four AI features use a structured prompt shape: role assignment, factual context block drawn from the database, then explicit output format instructions. Product summaries are capped at 3 to 5 sentences. Per ingredient explanations are required to be exactly 2 sentences. Both use a retry loop of up to 3 attempts that inspects the final character for sentence terminating punctuation. Incomplete responses are never cached, complete ones are hashed and cached in session state.

This approach grounds the model in database content before asking it to generate, reducing hallucination risk and producing consistently structured output.

## Prioritization (ESVF Framework)

Features were scored using the **Effort, Strategic Value, Business Value, Feasibility** framework:

```
ESVF Score = (Strategic Value + Business Value + Feasibility) / Effort
```

Each dimension is scored 1 to 5. Features with higher scores land in the near term horizon; lower scores get deferred.

## Data Sources

Every ingredient record is traceable back to at least one of the following primary regulatory or scientific references:

1. **European Commission Cosmetics Regulation (EC No 1223/2009)**. Over 1,300 banned or restricted cosmetic substances across Annexes II to VI.
2. **California Toxic Free Cosmetics Act (AB 2762)**. 24 prohibited chemicals including mercury, formaldehyde, lead, and several PFAS compounds.
3. **IARC Monograph Series**. Carcinogenicity classifications (Group 1, 2A, 2B, 3) from the International Agency for Research on Cancer.
4. **Cosmetic Ingredient Review (CIR) Expert Panel reports**. Peer reviewed safety assessments for cosmetic ingredients.
5. **FDA cosmetic ingredient guidance**. US federal baseline, expanding under MoCRA 2022.
6. **California Proposition 65**. OEHHA list of chemicals known to cause cancer or reproductive toxicity.

Where frameworks conflict, the more conservative classification is adopted, consistent with the precautionary principle applied in EU cosmetic regulation.

## Known Limitations

* **Database coverage**. 736 ingredients is substantial but does not cover the full universe of potentially harmful cosmetic chemicals. Ingredients outside the database return no results.
* **Very low resolution images**. Images below 400 pixels in height cannot be reliably recovered even with heavy path preprocessing. The app displays a low resolution warning in these cases.
* **Concentration blind**. The system does not account for ingredient concentration. An ingredient flagged as restricted above 0.5% is treated the same regardless of its actual concentration in the product.
* **Quantitative OCR benchmarking**. No labeled ground truth dataset for cosmetic ingredient OCR exists in the public domain. Performance was evaluated qualitatively across five representative image condition categories.
